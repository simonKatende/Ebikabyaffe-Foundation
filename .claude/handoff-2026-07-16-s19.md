## Bug-polish pass (same session, after the scaffold)

A full backend+frontend bug sweep was run over the new scaffold and the
recent (s16–s18) frontend surfaces. Five real issues found and fixed:

1. **`is_clan_officer()` had no pinned `search_path`** (migration 2) — a
   `security definer` function without `set search_path` is open to
   search-path hijacking and is flagged by Supabase's own database linter.
   Now `set search_path = public`.
2. **`verification_audit.actor_id` / `announcements.author_id` FKs would
   block user deletion** — plain `references auth.users` defaults to NO
   ACTION, so deleting any user who ever acted would fail. Both are now
   `on delete set null` (the audit trail survives; `actor_label` keeps the
   display name).
3. **`profiles.updated_at` would freeze at insert time** — added a
   `set_updated_at()` trigger (migration 1).
4. **`lib/supabase/client.ts` built a new client per call** — now memoized
   in a module variable; multiple GoTrueClient instances fight over the same
   auth storage key (Supabase warns about exactly this).
5. **LoginFlow rendered broken copy for unknown phone prefixes** — e.g.
   074/079 produced "…code to the **your network** number 0740 …".
   `detectNetwork` now returns `null` for unmapped prefixes and the
   confirmation sentence just omits the network name ("…code to the number
   0740 123 456…"). The demo prefix map itself was NOT grown (per the s17
   rule). Playwright-verified: MTN copy unchanged, 074 copy fixed, and the
   full OTP sign-in with an 074 number still lands on /profile.

Also checked and found NOT to be bugs: the exogamy offset math in
`mockMembers.ts` can never pick the member's own clan (offset range 1–55
mod 56); `EmpangoCountdown`'s render-time `Date.now()` can't hydration-
mismatch because it only renders in the logged-in dashboard, which never
SSRs (`isLoggedIn` starts false). ⚠️ Heads-up, not a bug: EmpangoCountdown's
target date is hardcoded `2026-07-31` — from 1 Aug it shows "0 days away"
until someone updates it for Empango 2027.

Since no live Supabase project exists yet, the migration fixes were made
in place (editing the two migration files) rather than as a third
migration — nothing has ever applied them.
