## Playwright note (new, worth knowing for future sessions)

No browser-automation tool (`chromium-cli` etc.) was available in this
environment. Used `npx --yes playwright --version` to confirm Playwright
itself resolves via `npx`, but running an actual script needs the module
importable — `npx -p playwright node script.js` did **not** work (module
resolution didn't pick it up from the npx cache). What worked: `cd web &&
npm install --no-save playwright` (pulls it into `web/node_modules`
without touching `package.json`/`package-lock.json` — verified clean via
`git status` afterward each time), then run the script from inside
`web/`. Chromium browser binaries were apparently already available/cached
— no separate `playwright install` step was needed. All throwaway
screenshot/verification scripts were deleted from `web/` after each use;
none should remain in the tree.
