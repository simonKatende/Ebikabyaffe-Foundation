## What was built this session

### `/roots` — Bataka Verification Funnel ✅

A full 4-step sequential funnel that takes a logged-out visitor through sign-in, then guides a logged-in member to select their clan branch, record their lineage, and request verification from their Omutaka.

#### New files

| File | Purpose |
|------|---------|
| `web/app/roots/page.tsx` | Route: shows `RootsGate` if logged out, `RootsContent` if logged in |
| `web/components/roots/RootsGate.tsx` | Sign-in prompt with 4 feature cards and "Sign in to begin →" CTA |
| `web/components/roots/RootsContent.tsx` | Container: 5-dot progress bar + state management for all 4 steps |
| `web/components/roots/RootsSteps.tsx` | All 4 step cards + sub-components (SigaStep, MutubaStep, LineageStep, VerificationStep) |

#### Step flow

```
Progress dots: Clan ● — Siga ○ — Mutuba ○ — Lineage ○ — Verified ○
               (always done)

Step 0 — Siga selection
  2×2 grid: Nkima 🐒 | Mmamba 🐟 | Ngabi 🦌 | I don't know ❓
  → on select: collapses to "✓ Done · tap to edit", unlocks Step 1

Step 1 — Mutuba (subclan) selection
  Dynamic options based on selected Siga:
    mmamba → Nkima, Kasimba, Nkerebwe
    nkima  → Lunyerekera, Kaboggoza, Nakayima
    ngabi  → Ngabi, Mbwa, Namugenyi
    unknown → skip message + "Continue to Lineage" button
  → on select: unlocks Step 2

Step 2 — Lineage form
  Fields: Full name, Father's name, Grandfather's name, Ssaza (select from amasaza[])
  Save button disabled until all 4 fields filled
  → on submit: unlocks Step 3

Step 3 — Verification request
  Summary table (Siga, Mutuba, Full name, Father, Grandfather, Ssaza)
  Royal-blue Bataka Council card
  "Request Verification from Omutaka →" CTA
  → on click: shows "Verification Pending ⏳" state
```

#### State (all in RootsContent)

```ts
const [step, setStep] = useState(0);            // 0-3: which step card is open
const [completed, setCompleted] = useState(-1); // highest completed step index
const [selectedSiga, setSelectedSiga] = useState<string | null>(null);
const [selectedMutuba, setSelectedMutuba] = useState<string | null>(null);
const [lineageData, setLineageData] = useState<LineageData | null>(null);
const [verificationPending, setVerificationPending] = useState(false);
```

`LineageData` type is exported from `RootsSteps.tsx`:
```ts
export type LineageData = {
  fullName: string;
  fatherName: string;
  grandfatherName: string;
  ssaza: string;
};
```

#### Step card unlock logic

```
isLocked   = index > completed + 1   // can't open at all
isOpen     = index === step && !isLocked  // currently showing content
isDone     = index <= completed      // completed (shows collapsed green)
```

User can tap a completed step header to re-open it for editing (sets `step = index`).
